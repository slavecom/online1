## 1. 전체 현황 분석

## 3. 문제 항목 분석
- 누락값이 있는 항목
- 오류 가능성이 있는 항목
- 이상값
- 중복 또는 표기 불일치
- 상태값과 실제 데이터가 맞지 않는 항목
## 4. 우선순위 분석
- 먼저 확인해야 할 항목
- 업무 영향도가 큰 항목
- 조치가 지연될 경우 문제가 커질 수 있는 항목
- 담당자 확인이 필요한 항목
## 5. 패턴 분석
- 반복적으로 나타나는 문제
- 특정 그룹에 집중된 문제
- 특정 상태나 유형에서 많이 발생하는 문제

- 추가 확인이 필요한 항목

③ 미완료 14건은 용접·검사·도장 후반 공정에 몰려 마감 부하 예상.
④ 결측·형식오류는 정리 완료, 남은 포인트는 조립·LNG 지연·계획일 정확도.
⑤ 담당자 4건·일부 진척률은 [가정]값이라 실제 확인 필요.

보고서 작성 — 두 개의 MD로 Word 만들기

데이터분석.md(사실)와 인사이트.md(해석·조치)를 근거로 DOCX를 생성합니다.

복사용 프롬프트 
첨부한 데이터분석.md 파일과 인사이트.md 파일을 바탕으로
현업 보고용 Word 보고서 DOCX 파일을 생성해줘.
보고서 파일명은 다음과 같이 해줘.
공정현황_데이터검토_보고서.docx
보고서 구성은 다음과 같이 작성해줘.
1. 보고 제목
2. 보고 목적
3. 데이터 검토 범위
4. 주요 검증 결과
5. 데이터 분석 결과
6. 핵심 인사이트
7. 조치 필요사항
8. 추가 확인사항
9. 향후 개선 제안
작성 조건:
- 결론을 먼저 작성해.
- 보고 문체는 간결하고 객관적으로 작성해.
- 데이터분석.md의 내용은 사실 근거로 사용해.
- 인사이트.md의 내용은 해석과 조치사항으로 사용해.
- 확인되지 않은 내용은 [확인필요]라고 표시해.
- 데이터에 없는 원인을 새로 만들지 마.
- 표가 필요한 부분은 표로 정리해.
- 상급자 보고용으로 너무 길지 않게 작성해.

왜 MD 파일을 먼저 만들면 좋은가

보고서(DOCX)로 바로 가지 않고, 데이터분석.md · 인사이트.md를 먼저 만드는 이유입니다.

🤖AI가 읽기 쉬운 문서

MD는 표식이 단순한 순수 텍스트라 AI가 오해 없이 읽고 다시 활용하기 좋습니다.

🔁재사용 쉽다

MD 하나로 Word·PPT·Excel·
대시보드. 내용은 한 곳에서 관리.

✂수정·검수 편함

텍스트라 고친 곳이 한눈에.
문장 단위 검토에 좋습니다.

🧾이력 추적

"어떤 사실→이 결론"이
중간 근거로 보존됩니다.

❌ MD 없이 바로 Word로 가면

사실·해석·형식이 뒤섞이고, 고칠 때마다 문서를 통째로 다시 손봐야 하며,
어떤 근거로 그 결론이 나왔는지 남지 않습니다.

✅ MD를 먼저(우선) 만들면
